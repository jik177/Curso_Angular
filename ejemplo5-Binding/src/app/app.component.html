BINDING

CONSTRUCCIONES EN ANGULAR -data binding-


1º String Interpolation: ( )typeScript ->html

Esta primera fase de construccion, presentada con anterioridad, sin otra caracteristica desarrollar o presentar una informacion(variables, arrays, colecciones...) generando visualizaciones directas en html
Sintaxis:

  • He definido la variable {{title}}
  • Concatenacion de variables: {{nombre + ' ' + 'Primer nombre' + ' y ' + apellido }}
  • Operador logico suma: {{ (3+7+15) + nombre}}

2º Event Binding ( )typeScript <- html

Escuchas o ejecuciones de un evento desarrolado en ts(js) desde html generará acciones o construcciones de estructura(estilos, elementos, eventos..)

  • Desarrollo habitual de script, ejecutando acciones que realiza el usuario
  • Ejemplo de mensajes como interaccion o UX
    • Inyectar una libreria SweetAler2 en el evento
    • Importar dicho frameWork dentro de este componente

3º Property Binding ( )typeScript <= Html

Informacion generada desde el plano html, partiendo de la accion del usuario o de la propia ejecucion del motor de Doctype(html). Desde la accion de typescript definido, redefina, o propuesta de valor a a ejecutar

4º Two way Data Binding ( )typeScript <=> Html

Ejecuciones de comunicacion e interaccion en dos sentidos. Cliente (Servicio) a typeScript y viceversa
Cambios en la construccion de html y viceversa adoptando los cambios pero no sustancialmente (Cambios subitos).



5º Event: .this: Binding(-Clases-)

Un recurso de llamada a los objetos dentro de html, tendremos que invocar a la propiedad del objetos no podemos trabajar direcctamente sobre el elemento. Como referenciamos esa llamada

    Sintaxis
  • this.propiedad
  • Valor: Determinado por como invoca a la funcion o la propiedad
  • Otras configuraciones del .this
    • Global: Fuera de cualquier funcion siempre referenciará al objeto (global)
